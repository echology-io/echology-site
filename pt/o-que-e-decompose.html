O Que E o
Decompose?

Classificacao deterministica de texto para agentes de IA. Sem LLM, sem probabilidade, sem alucinacao. Texto entra, estrutura classificada sai.

A resposta curta

Decompose e uma biblioteca open-source em Python que classifica texto de forma deterministica. Ela recebe texto bruto e retorna unidades semanticas estruturadas com scores de autoridade, risco, atencao e entidades. Usa pattern matching baseado em regex no lugar de machine learning, o que significa que a mesma entrada sempre produz a mesma saida. Sem dados de treinamento, sem pesos de modelo, sem distribuicao de probabilidade. Apenas pattern matching contra estruturas linguisticas conhecidas.

Instale pelo PyPI:

pip install decompose-mcp

Use com uma unica chamada de funcao:

from decompose import decompose_text result = decompose_text("O empreiteiro devera cumprir a ASTM C150 para todo cimento portland.") # Retorna unidades estruturadas com: # - authority: "mandatory" (detectou "devera") # - risk: "compliance" # - attention: 4.2 # - entities: ["ASTM C150"] # - irreducible: True

O que ele classifica

O Decompose quebra o texto em unidades semanticas e pontua cada uma em quatro dimensoes.

Autoridade

Detecta se a linguagem e obrigatoria ("devera," "deve," "e obrigado a"), diretiva ("convem," "e esperado que"), ou permissiva ("pode," "e permitido"). E assim que voce separa obrigacoes de sugestoes em qualquer documento.

Risco

Categoriza o conteudo por tipo de risco: seguranca, financeiro, conformidade, juridico ou informacional. Uma clausula sobre requisitos de carga e seguranca. Uma clausula sobre condicoes de pagamento e financeira. Um paragrafo descrevendo historico do projeto e informacional.

Atencao

Atribui um score de prioridade de 0 a 10 com base no sinal combinado de autoridade, risco e densidade de entidades. Unidades de alta atencao sao as que mais importam. Unidades de baixa atencao podem ser ignoradas ou despriorizada.

Entidades

Extrai referencias a normas (ASTM, ACI, ISO), datas, valores monetarios, percentuais e outros dados estruturados. Sao os fatos concretos embutidos em prosa que sistemas downstream precisam.

Tambem marca conteudo irredutivel: texto que nao pode ser parafraseado sem perder significado. Valores monetarios, referencias de codigo, valores-limite, linguagem juridica literal. Sao unidades que qualquer sistema downstream precisa preservar exatamente como escritas.

Por que classificacao deterministica importa

A maioria da classificacao de texto hoje usa machine learning. Voce treina um modelo com dados rotulados e ele prediz categorias com alguma probabilidade. Isso funciona pra varias tarefas, mas introduz um problema fundamental: a mesma entrada pode produzir saidas diferentes. Atualizacoes de modelo mudam predicoes. Configuracoes de temperatura introduzem aleatoriedade. Scores de confianca variam entre execucoes.

O Decompose elimina isso completamente. A classificacao dele e deterministica. O mesmo texto sempre produz a mesma saida. Todas as vezes. Em qualquer maquina. Sem pesos de modelo, sem GPU, sem chamada de API.

Isso importa por tres razoes:

  • Ground truth para pipelines de IA. Quando voce usa o Decompose como etapa de pre-processamento antes de um LLM, o LLM recebe input estruturado e classificado em vez de texto bruto. A camada de classificacao e estavel, auditavel e reproduzivel. A camada probabilistica acima pode mudar sem quebrar a fundacao.
  • Auditabilidade. Toda decisao de classificacao pode ser rastreada ate um pattern match especifico. Nao tem caixa preta. Quando uma unidade e classificada como "obrigatoria, seguranca, atencao 7.2," voce consegue inspecionar exatamente quais padroes dispararam essa classificacao.
  • Velocidade. O Decompose roda em milissegundos, nao segundos. Nao tem inferencia de modelo, nao tem latencia de API, nao tem custo de token. A classificacao acontece na velocidade de execucao de regex.

Casos de uso

Pre-processamento para pipelines de LLM

Em vez de mandar documentos brutos para um LLM, decomponha primeiro. Filtre por score de atencao. Mande so as unidades de alta prioridade pro modelo. Isso reduz custo de tokens, melhora a qualidade das respostas e da ao LLM contexto estruturado em vez de ruido.

Triagem de documentos

Quando centenas de documentos chegam (especificacoes, contratos, relatorios, submissoes), decomponha todos. Ordene por score de atencao. Direcione conteudo obrigatorio para revisores, pule boilerplate informacional. Uma especificacao de 200 paginas vira 30 paginas de material que realmente precisa de atencao humana.

Filtragem de conteudo para RAG

Retrieval-augmented generation funciona melhor quando os chunks no seu vector store sao unidades semanticas classificadas, e nao divisoes arbitrarias de texto. O Decompose da a cada chunk uma estrutura: autoridade, risco, atencao, entidades. Seu sistema de retrieval pode filtrar por essas dimensoes antes que o LLM veja o conteudo.

Analise de conformidade e risco

Extraia todo requisito obrigatorio de um documento regulatorio. Sinalize toda obrigacao financeira de um contrato. Traga a tona toda clausula critica de seguranca de uma especificacao. O Decompose faz isso sem treinar um modelo customizado, porque os padroes linguisticos que sinalizam obrigacao e risco sao estruturais, nao estatisticos.

Ferramenta MCP para agentes de IA

O Decompose roda como servidor de ferramentas Model Context Protocol (MCP). Agentes de IA podem chamar o Decompose como ferramenta, recebendo dados de classificacao estruturados sobre os quais podem raciocinar. Isso da aos agentes uma camada de percepcao: eles conseguem ver o que um documento contem antes de decidir o que fazer com ele.

# Rodar como servidor de ferramentas MCP python -m decompose --serve # Ou usar direto no codigo do seu agente from decompose import decompose_text # Decompor antes de enviar ao LLM result = decompose_text(document_text) high_priority = [u for u in result["units"] if u["attention"] >= 3.0] # Enviar so o que importa pro modelo for unit in high_priority: response = llm.analyze(unit["text"], context=unit["metadata"])

Como funciona por baixo dos panos

O Decompose usa expressoes regulares compiladas para detectar padroes linguisticos que sinalizam autoridade, risco e importancia estrutural. Esses padroes sao baseados em como obrigacoes, permissoes e requisitos sao expressos na escrita tecnica e profissional.

O pipeline de classificacao tem tres estagios:

  1. Segmentacao. O texto bruto e dividido em unidades semanticas nos limites de sentencas e clausulas, preservando o contexto.
  2. Pattern matching. Cada unidade e testada contra bibliotecas de padroes para linguagem de autoridade, indicadores de risco, tipos de entidade e marcadores de irredutibilidade.
  3. Scoring. Os pattern matches sao agregados nas quatro dimensoes de classificacao (autoridade, risco, atencao, entidades) e um score geral de atencao e computado.

Sem etapa de treinamento. Sem arquivo de modelo. Sem dependencia de servicos externos. As bibliotecas de padroes sao incluidas no pacote e executam na velocidade nativa de regex.
