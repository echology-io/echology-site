What Is
Decompose?

Deterministic text classification for AI agents. No LLM, no probability, no hallucination. Text in, classified structure out.

The short answer

Decompose is an open-source Python library that classifies text deterministically. It takes raw text and returns structured semantic units scored by authority, risk, attention, and entities. It uses regex-based pattern matching instead of machine learning, which means the same input always produces the same output. No training data, no model weights, no probability distribution. Just pattern matching against known linguistic structures.

Install it from PyPI:

pip install decompose-mcp

Use it in one function call:

from decompose import decompose_text result = decompose_text("The contractor shall comply with ASTM C150 for all portland cement.") # Returns structured units with: # - authority: "mandatory" (detected "shall") # - risk: "compliance" # - attention: 4.2 # - entities: ["ASTM C150"] # - irreducible: True

What it classifies

Decompose breaks text into semantic units and scores each one across four dimensions.

Authority

Detects whether language is mandatory ("shall," "must," "is required to"), directive ("should," "is expected to"), or permissive ("may," "is permitted to"). This is how you separate obligations from suggestions in any document.

Risk

Categorizes content by risk type: safety, financial, compliance, legal, or informational. A clause about load-bearing requirements is safety. A clause about payment terms is financial. A paragraph describing project history is informational.

Attention

Assigns a 0 to 10 priority score based on the combined signal of authority, risk, and entity density. High-attention units are the ones that matter most. Low-attention units can be skipped or deprioritized.

Entities

Extracts standards references (ASTM, ACI, ISO), dates, dollar amounts, percentages, and other structured data points. These are the concrete facts embedded in prose that downstream systems need.

It also flags irreducible content: text that cannot be paraphrased without losing meaning. Dollar amounts, code references, threshold values, verbatim legal language. These are units that any downstream system must preserve exactly as written.

Why deterministic classification matters

Most text classification today uses machine learning. You train a model on labeled data, and it predicts categories with some probability. This works for many tasks, but it introduces a fundamental problem: the same input can produce different outputs. Model updates change predictions. Temperature settings introduce randomness. Confidence scores shift across runs.

Decompose eliminates this entirely. Its classification is deterministic. The same text always produces the same output. Every time. On every machine. Without model weights, without GPU, without an API call.

This matters for three reasons:

  • Ground truth for AI pipelines. When you use decompose as a preprocessing step before an LLM, the LLM receives structured, classified input instead of raw text. The classification layer is stable, auditable, and reproducible. The probabilistic layer above it can change without breaking the foundation.
  • Auditability. Every classification decision can be traced to a specific pattern match. There is no black box. When a unit is classified as "mandatory, safety, attention 7.2," you can inspect exactly which patterns triggered that classification.
  • Speed. Decompose runs in milliseconds, not seconds. There is no model inference, no API latency, no token cost. Classification happens at the speed of regex execution.

Use cases

Preprocessing for LLM pipelines

Instead of sending raw documents to an LLM, decompose them first. Filter by attention score. Send only high-priority units to the model. This reduces token cost, improves response quality, and gives the LLM structured context instead of noise.

Document triage

When hundreds of documents arrive (specifications, contracts, reports, submissions), decompose them all. Sort by attention score. Route mandatory content to reviewers, skip informational boilerplate. A 200-page specification becomes 30 pages of material that actually requires human attention.

Content filtering for RAG

Retrieval-augmented generation works better when the chunks in your vector store are classified semantic units instead of arbitrary text splits. Decompose gives each chunk a structure: authority, risk, attention, entities. Your retrieval system can then filter by these dimensions before the LLM ever sees the content.

Compliance and risk analysis

Extract every mandatory requirement from a regulatory document. Flag every financial obligation in a contract. Surface every safety-critical clause in a specification. Decompose does this without training a custom model, because the linguistic patterns that signal obligation and risk are structural, not statistical.

MCP tool for AI agents

Decompose runs as a Model Context Protocol (MCP) tool server. AI agents can call decompose as a tool, receiving structured classification data they can reason about. This gives agents a perception layer: they can see what a document contains before deciding what to do with it.

# Run as an MCP tool server python -m decompose --serve # Or use directly in your agent code from decompose import decompose_text # Decompose before sending to LLM result = decompose_text(document_text) high_priority = [u for u in result["units"] if u["attention"] >= 3.0] # Only send what matters to the model for unit in high_priority: response = llm.analyze(unit["text"], context=unit["metadata"])

How it works under the hood

Decompose uses compiled regular expressions to detect linguistic patterns that signal authority, risk, and structural importance. These patterns are based on how obligations, permissions, and requirements are expressed in professional and technical writing.

The classification pipeline has three stages:

  1. Segmentation. Raw text is split into semantic units at sentence and clause boundaries, preserving context.
  2. Pattern matching. Each unit is tested against pattern libraries for authority language, risk indicators, entity types, and irreducibility markers.
  3. Scoring. Pattern matches are aggregated into the four classification dimensions (authority, risk, attention, entities) and an overall attention score is computed.

No training step. No model file. No dependency on external services. The pattern libraries are included in the package and execute at native regex speed.

Get started

# Install pip install decompose-mcp # Classify any text from decompose import decompose_text result = decompose_text(open("document.md").read()) # Inspect the output for unit in result["units"]: print(unit["authority"], unit["risk"], unit["attention"], unit["text"][:80])
